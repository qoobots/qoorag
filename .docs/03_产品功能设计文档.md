## 2. 产品功能架构（功能地图）

```
qoorag 平台
├── A. 系统管理（系统管理员）
│   ├── 用户管理（增/停/改）
│   ├── 角色管理（增/改、菜单与权限）
│   ├── 用户-角色关系
│   ├── 资源池管理（LLM/Embedding/Rerank/向量库 接入与纳管）
│   └── 审计日志查看
├── B. 知识库管理（知识管理员）
│   ├── 知识库 CRUD（命名、六层配置选择）
│   ├── 文档管理（上传/解析/分块/向量化/入库）
│   ├── 检索权限管理（RBAC 授权：角色/部门/用户/外部）
│   ├── API Key 管理（签发/轮换/吊销/限流配置）
│   └── 删除与保留期（软删/恢复/物理清理）
├── C. 检索与问答（授权用户/API）
│   ├── 知识库内问答（检索+生成+来源引用）
│   ├── 多轮对话上下文
│   └── 流式输出
├── D. 对外 API（OpenAI 兼容）
│   ├── /retrieve 检索
│   └── /chat 问答
└── E. 合规与可观测（贯穿）
    ├── 操作审计日志
    ├── 问答留痕（QaTrace）
    ├── 数据分级字段
    └── 调用量/用量统计
```

## 3. 功能模块详细设计

### 3.1 系统管理（系统管理员）

**3.1.1 用户管理**

- 新增用户：账号、姓名、初始密码（强制首次改密）、归属租户(tenant_id)、状态（启用/停用）。
- 编辑/停用：停用后禁止登录，历史审计与问答留痕保留。
- 不开放自助注册；无公网注册入口。

**3.1.2 角色管理**

- 新增/编辑角色，定义可见菜单与权限点（MVP 先用两默认角色，支持扩展）。
- 用户-角色分配/收回。

**3.1.3 资源池管理（核心：集中纳管，禁止外部自带）**

- 系统管理员统一接入并维护：
  - LLM（生成）端点与密钥；
  - Embedding 模型端点与密钥；
  - Rerank 模型（可选）端点；
  - 向量库实例（pgvector 等）。
- 知识管理员**只能从资源池选择**，不得自行接入外部模型/外部向量库（架构上保障数据不出域，见 4.7、4.8）。
- 资源池对全体知识管理员可见可选；MVP 资源池条目来自配置文件/初始化，后续可界面化管理。

**3.1.4 审计日志查看**

- 查看全平台操作审计（登录、权限变更、知识库增删改、配置变更、Key 操作等）。
- 只读、不可删除/篡改，支持按时间/操作人/对象筛选。

### 3.2 知识库管理（知识管理员）

**3.2.1 知识库创建**

- 必填：名称、描述、归属租户(tenant_id)、数据分级（公开/内部/机密/绝密，可配置）。
- 六层配置：从资源池选择 数据接入策略 / 分块策略 / Embedding / 向量库 / 检索方式 / Rerank / LLM / Prompt 模板（MVP 提供一套默认配置，可后续调）。
- 创建后默认**私有**。

**3.2.2 文档管理（数据接入→处理→索引）**

- 上传格式（MVP）：PDF、Markdown、TXT（后续扩展 Word/PPT/Excel/HTML、数据源连接器）。
- 处理流水线：解析 → 分块（固定/语义/标题层级，MVP 固定长度+可配） → Embedding 向量化 → 写入向量库（带 tenant_id、kb_id）。
- 状态可见：上传中/解析中/向量化中/已入库/失败（失败可重试）。
- 文档级元数据：来源、上传时间、所属知识库、数据分级。

**3.2.3 检索与问答（知识库内）**

- 用户输入问题 → 向量检索（+关键词 BM25，MVP 可先纯向量） → （可选 Rerank） → 召回 Top-K 片段 → LLM 生成答案并标注来源引用。
- 多轮对话上下文；流式输出；无答案时明确告知（幻觉抑制）。

**3.2.4 检索权限管理（RBAC 共享）**

- 授权对象：内部角色 / 部门 / 具体用户；经认证的外部用户（外部凭证）。
- 权限粒度：检索/问答（只读）。管理权不随授权转移。
- 默认私有；授权/收回全程留痕。
- 与数据分级联动：高密级库仅授权且具相应密级权限者可检索。

**3.2.5 删除与生命周期（软删+保留期）**

- 删除进入软删除（标记 deleted_at，暂停检索）。
- 保留期内可恢复；过期由定时任务物理清理**业务数据**（文档/分块/向量）。
- **审计日志与问答留痕独立保留**，不随删库删除（等保取证）。
- 清理受 tenant_id 约束，不影响其他租户/库。

### 3.3 资源池（系统管理员纳管，详见 3.1.3）

- 两层资源模型：平台资源池（复用+安全合规） + 知识库独立配置（隔离+灵活）。
- MVP 资源池通过配置/初始化提供；知识库在池内自由编排、互不干扰。

### 3.4 检索与问答（授权用户视角）

- 见 3.2.3。普通员工/外部用户仅能在被授权知识库内检索/问答。

### 3.5 对外 API（OpenAI 兼容）

**3.5.1 鉴权**

- 按知识库签发 API Key；调用凭 Key 访问**对应知识库**（继承检索权限与 tenant_id 隔离）。
- Key 仅展示一次、加密存储；支持轮换/吊销；传输 TLS。

**3.5.2 接口（MVP 骨架）**

- `POST /api/v1/retrieve`：检索召回片段。
- `POST /api/v1/chat`：检索+生成问答。
- 风格对齐 OpenAI 兼容（后续完善请求/响应契约，见 04.4 接口设计）。

**3.5.3 限流**

- 按 Key 维度速率限制（每分钟/每小时请求数与 Token 上限），非配额计费；策略由系统管理员配置。
- 每次调用记录 Key、知识库、调用方、时间、用量，纳入审计与问答留痕。

### 3.6 合规与可观测（贯穿全平台）

**3.6.1 审计（Audit）**

- 操作审计：登录、权限变更、知识库增删改、配置变更、Key 操作等，记录操作人/时间/对象/前后值。
- 全链路可追溯：检索与生成过程（模型、召回片段、Prompt、输出、参数）可回溯。
- 防篡改：只读、不可删、定期归档。

**3.6.2 留痕（Traceability）**

- 问答留痕（QaTrace）：问题、召回来源、答案、模型与参数、时间戳、用户身份。
- 知识库操作留痕：上传/解析/向量化/配置变更全程留痕。
- 留存周期：满足等保（一般≥6 个月），到期归档/安全销毁。

**3.6.3 数据分级**

- 知识库/文档标注敏感级别；分级访问控制；PII 脱敏（MVP 仅保留字段，脱敏/密评后置）。

## 4. 关键业务流程（用例）

### 4.1 系统初始化与开通

```
[系统管理员] 首次登录 admin → 强制改密
    → 创建用户（如 知识管理员 A）并分配角色
    → 维护资源池（LLM/Embedding/向量库）
```

### 4.2 知识库搭建到问答闭环

```
[知识管理员] 登录 RAG 管理页
    → 创建知识库（选默认六层配置，默认私有）
    → 上传文档（PDF/MD/TXT）
    → 系统自动：解析→分块→向量化→入库
    → 授权检索对象（角色/部门/用户/外部）
    → 在库内问答（检索+生成+来源引用）
```

### 4.3 对外 API 调用

```
[知识管理员] 签发 API Key → 配置限流
[开发者] 持 Key 调用 /api/v1/chat
    → 拦截器校验 Key→定位知识库→校验 tenant_id 隔离与检索权限
    → 检索+生成 → 返回答案（带来源）
    → 调用记录写入审计与 QaTrace
```

### 4.4 知识库删除与合规留存

```
[知识管理员] 删除知识库 → 软删除（暂停检索）
    → 保留期内可恢复
    → 过期定时任务物理清理业务数据（文档/分块/向量）
    → 审计日志 + 问答留痕 独立保留（不删）
```

## 5. 数据实体概要

核心实体（均带 `tenant_id` 与 `deleted_at`，完整表结构与字段见 04 §9 数据架构）：

| 实体 | 说明 |
| --- | --- |
| Tenant | 租户/组织单元 |
| User | 用户账号 |
| Role | 角色 |
| KnowledgeBase | 知识库（含六层配置、数据分级） |
| KbPermission | 知识库检索权限授权记录 |
| Document | 文档元信息 |
| Chunk | 文档分块 |
| VectorData | 向量数据（embedding，带 tenant_id/kb_id） |
| ApiKey | 知识库 API Key（加密存储、限流配置） |
| AuditLog | 操作审计日志（独立留存） |
| QaTrace | 问答留痕（独立留存） |

关系要点：User—Role 多对多；KnowledgeBase 归属 Tenant；KbPermission 关联 KB 与授权对象；Document→Chunk→VectorData 级联；ApiKey 绑定 KB；AuditLog/QaTrace 独立、不随删库删除。
