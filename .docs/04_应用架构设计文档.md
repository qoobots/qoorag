## 2. 应用架构总览

### 2.1 应用组合

| 应用/模块 | 对应业务能力 | 主要消费者 |
| --- | --- | --- |
| 系统管理应用 | A. 系统管理 | 系统管理员 |
| 知识库管理应用 | B. 知识库管理 | 知识管理员 |
| 检索问答应用 | C. 检索与问答 | 授权用户 |
| 对外 API 应用 | D. 对外 API | AI 应用开发者 / 外部系统 |
| 合规可观测应用 | E. 合规与可观测 | 系统管理员 / 安全合规 |
| 认证鉴权（横切） | 登录 / API Key / 会话 | 全部应用 |

### 2.2 应用交互与集成

```
[浏览器/调用方]
   │
   ├─ 管理界面 → 系统管理 / 知识库管理 / 检索问答 / 合规可观测 应用
   │              （Session Token 鉴权，注入 tenant_id + 角色）
   │
   └─ 对外 API → 对外 API 应用（API Key 鉴权）
                   → 继承绑定 KB 的 tenant_id 隔离与 RBAC 检索权限
                   → 复用 检索问答应用 的检索+生成能力
所有应用 → 合规可观测应用（写入 AuditLog / QaTrace）
所有应用 → 数据架构（03）的持久化与隔离
```

## 3. 应用模块详细设计

### 3.1 系统管理应用（系统管理员）

- **用户管理**：新增用户（账号、姓名、初始密码强制首次改密、归属 tenant_id、状态）；编辑/停用（停用后禁止登录，历史审计与问答留痕保留）；不开放自助注册。
- **角色管理**：新增/编辑角色，定义可见菜单与权限点（MVP 先用两默认角色，支持扩展）；用户-角色分配/收回。
- **资源池管理（核心）**：系统管理员统一接入维护 LLM / Embedding / Rerank（可选）/ 向量库；知识管理员只能从资源池选择，不得自行接入外部资源（保障数据不出域）；MVP 资源池来自配置/初始化，后续界面化。
- **审计日志查看**：查看全平台操作审计，只读、不可删除/篡改，支持按时间/操作人/对象筛选。

### 3.2 知识库管理应用（知识管理员）

- **知识库创建**：必填名称、描述、归属 tenant_id、数据分级；六层配置从资源池选择（MVP 提供一套默认配置）；创建后默认私有。
- **文档管理（数据接入→处理→索引）**：上传 PDF/MD/TXT → 解析 → 分块（固定长度+可配） → Embedding 向量化 → 写入向量库（带 tenant_id、kb_id）；状态可见（上传中/解析中/向量化中/已入库/失败，失败可重试）；文档级元数据（来源、上传时间、所属库、数据分级）。
- **检索权限管理（RBAC 共享）**：授权对象为内部角色/部门/具体用户/经认证外部用户；粒度仅检索/问答（只读）；默认私有；授权/收回全程留痕；与数据分级联动。
- **API Key 管理**：签发/轮换/吊销/限流配置；Key 仅展示一次、加密存储。
- **删除与生命周期**：软删+保留期+恢复+过期物理清理（见 03 §7）。

### 3.3 检索问答应用（授权用户视角）

- 用户输入问题 → 向量检索（+关键词 BM25，MVP 可先纯向量） → （可选 Rerank） → 召回 Top-K 片段 → LLM 生成答案并标注来源引用。
- 多轮对话上下文；流式输出；无答案时明确告知（幻觉抑制）。
- 普通员工/外部用户仅能在被授权知识库内检索/问答。

### 3.4 对外 API 应用（OpenAI 兼容）

- **鉴权**：按知识库签发 API Key；调用凭 Key 访问对应知识库（继承检索权限与 tenant_id 隔离）；传输 TLS。
- **接口（MVP 骨架）**：
  - `POST /api/v1/retrieve`：检索召回片段。
  - `POST /api/v1/chat`：检索+生成问答。
  - 风格对齐 OpenAI 兼容（后续完善请求/响应契约，见 05.4 接口设计）。
- **限流**：按 Key 维度速率限制（每分钟/每小时请求数与 Token 上限），非配额计费；策略由系统管理员配置；每次调用记录 Key、知识库、调用方、时间、用量，纳入审计与问答留痕。

### 3.5 合规可观测应用（贯穿全平台）

- **审计（Audit）**：操作审计（登录、权限变更、KB 增删改、配置变更、Key 操作），记录操作人/时间/对象/前后值；全链路可追溯；防篡改只读、定期归档。
- **留痕（Traceability）**：问答留痕（QaTrace：问题、召回来源、答案、模型与参数、时间戳、用户身份）；知识库操作留痕；留存满足等保（一般≥6 个月），到期归档/安全销毁。
- **数据分级**：知识库/文档标注敏感级别；分级访问控制；PII 脱敏（MVP 仅保留字段，脱敏/密评后置）。

### 3.6 认证鉴权（横切）

- 登录校验 BCrypt 密码，签发 Session Token（注入 SecurityContext：用户/角色/tenant_id）。
- API Key 校验 `key_hash`（SHA-256），定位绑定知识库，继承 tenant_id 隔离与 RBAC 检索权限。
- 白名单：登录接口放行，其余需鉴权；CORS 由 WebConfig 配置。

## 4. 应用接口与集成

### 4.1 对外 API 契约（骨架）

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| POST | `/api/v1/retrieve` | 检索召回片段（带 Key 鉴权） |
| POST | `/api/v1/chat` | 检索+生成问答（带 Key 鉴权） |

> 完整请求/响应契约、错误码规范见 05.4 接口设计（`.docs/05.4_接口设计.md` + 机读规范 `.docs/openapi.yaml`）。

### 4.2 内部服务边界

- 应用模块间通过 Service 边界解耦（见 05 §5 模块拆分）：系统管理、知识库管理、认证鉴权、对外 API、合规可观测、初始化。
- 检索/生成、文档处理是可独立成服务的候选边界（未来微服务化）。

---

## 5. 应用部署单元

采用**分层单体（Modular Monolith）+ 独立微服务边界**风格（详见 05 §2）：

- 当前 MVP 以单一 Spring Boot 应用承载全部应用模块（部署简单、私有化友好），内部按业务能力清晰分层与分包。
- 保持"独立微服务"定位：未来可将检索/生成、文档处理、API 网关拆分为独立服务，当前包结构已为拆分预留边界。
