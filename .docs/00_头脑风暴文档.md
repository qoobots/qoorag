## 1. 一句话愿景

打造一个**开箱即用、可自托管、模块化可扩展**的开源 RAG（检索增强生成）系统，让个人开发者和企业都能低成本地把私有知识接入大模型。

---

## 2. 我们想解决什么问题

- 大模型缺乏私有 / 实时 / 领域知识，容易"一本正经地胡说"。
- 现有 RAG 方案要么太重（企业级、难部署），要么太轻（demo 级、不可用于生产）。
- 用户搭建 RAG 需要拼接：文档解析、切分、向量化、检索、重排、生成……链路长、门槛高。
- 缺乏一个"约定优于配置"、能快速跑通全流程、又能逐步深入定制的开源项目。

---

## 3. 目标用户画像

| 用户类型 | 核心诉求 | 使用场景 |
| --- | --- | --- |
| 个人开发者 / 极客 | 快速搭建私人知识库问答 | 本地文档、笔记、代码库问答 |
| 中小团队 | 内部知识库、客服助手 | 产品文档、FAQ、工单辅助 |
| 企业技术团队 | 可控、可私有化、可二次开发 | 内部知识管理、智能助手中台 |
| AI 应用开发者 | 作为底层能力集成到自己产品 | 提供检索 / 问答 API |

## 4. 核心功能构想（发散）

### 4.1 数据接入层
- 多格式文档解析：PDF、Word、Markdown、HTML、TXT、PPT、Excel。
- 数据源连接器：本地文件、URL 抓取、Notion、Confluence、Git 仓库、数据库。
- 增量同步与定时更新。
- OCR 支持（扫描件、图片中的文字）。

### 4.2 处理与索引层
- 智能分块（Chunking）：固定长度、语义分块、按标题层级分块。
- 多种 Embedding 模型支持（OpenAI、BGE、M3E、本地模型等，可插拔）。
- 向量数据库可选：Milvus、Qdrant、pgvector、Chroma、Faiss。
- 元数据管理与过滤（来源、时间、权限标签）。

### 4.3 检索层
- 向量检索 + 关键词检索（BM25）混合检索。
- 重排（Rerank）模型接入。
- 多路召回、结果去重与合并。
- 引用溯源（答案标注来源片段）。

### 4.4 生成层
- 多 LLM 支持：OpenAI、Claude、通义、DeepSeek、本地 Ollama 等，统一抽象接口。
- Prompt 模板管理与可视化调试。
- 流式输出、多轮对话上下文。
- 幻觉抑制策略（无答案时明确告知）。

### 4.5 应用与交互层
- 开箱即用的 Web 管理后台（知识库管理、上传、调试）。
- 内置对话 UI。
- 完整的 RESTful / OpenAI 兼容 API。
- SDK（Python / JS）。

### 4.6 运维与可观测
- 检索质量评估工具（召回率、命中率）。
- 调用日志、Token 统计、成本分析。
- Docker / K8s 一键部署。

## 5. 差异化亮点（怎么和别人不一样）

- **真·开箱即用**：一条命令跑通，默认配置即可用。
- **全链路可插拔**：Embedding、向量库、LLM、Rerank 每一环都能替换。
- **本地优先 / 隐私优先**：支持全本地部署，数据不出内网。
- **轻量但可扩展**：从单机 demo 平滑升级到生产集群。
- **可观测的 RAG**：内置检索评估与调试能力，不做"黑盒魔法"。

## 6. 技术选型初步想法（待验证）

- 后端：Python（FastAPI）为主，兼顾 AI 生态。
- 前端：React / Vue + 现代 UI 组件库。
- 向量库：默认 Chroma / pgvector（轻量），可切换 Milvus / Qdrant。
- 任务队列：Celery / Redis（处理异步解析与索引）。
- 部署：Docker Compose 起步，后续 Helm Chart。

---

## 7. MVP 范围建议（第一个可用版本）

聚焦"最短可用闭环"：

1. 上传文档（PDF / Markdown / TXT）。
2. 自动解析 → 分块 → 向量化 → 入库。
3. 提供问答接口（向量检索 + LLM 生成 + 来源引用）。
4. 一个简单的 Web 界面完成上传与问答。
5. Docker Compose 一键启动。

**先不做**：多数据源连接器、复杂权限、集群、评估平台等。

---

## 8. 潜在风险与挑战

- 文档解析质量参差（尤其是复杂 PDF / 表格）。
- 检索效果依赖 Embedding 与分块策略，需要大量调优。
- 多模型 / 多向量库抽象可能带来复杂度，需平衡灵活性与简洁性。
- 开源社区运营与文档质量决定项目能否被采用。
- 与 Dify、RAGFlow、FastGPT 等成熟项目的竞争与差异化。
