## 2. 业务背景与定位

### 2.1 业务驱动

企业大量私有知识（制度、规范、SOP、项目资料）散落在文档与人员脑中，检索低效、易泄露、难合规。qoorag 以可控、合规、低成本的方式把内部私有知识接入大模型，让"自然语言问、精准答、可追溯"成为企业基础能力。

### 2.2 一句话定位

**qoorag** 是一个企业级、可私有化部署、模块化可扩展的开源 RAG（检索增强生成）平台：支持多用户管理，每位用户可管理多个相互隔离的知识库，每个知识库拥有独立且可插拔的完整 RAG 链路。

### 2.3 产品形态

- **平台形态**：多用户 × 多知识库，用户间数据隔离，知识库间链路配置独立。
- **独立子系统**：作为"企业智能化"工程中自治子系统，通过标准 API 对外提供检索/问答能力，与统一门户/智能体平台/数据中台松耦合。
- **私有化优先**：全栈可部署于企业内网，知识数据不出域。

## 3. 业务能力地图

业务能力按"系统管理 / 知识库管理 / 检索问答 / 对外 API / 合规可观测"五级组织：

```
qoorag 平台（业务能力地图）
├── A. 系统管理（系统管理员）
│   ├── 用户管理（增/停/改）
│   ├── 角色管理（增/改、菜单与权限）
│   ├── 用户-角色关系
│   ├── 资源池管理（LLM/Embedding/Rerank/向量库 接入与纳管）
│   └── 审计日志查看
├── B. 知识库管理（知识管理员）
│   ├── 知识库 CRUD（命名、六层配置选择）
│   ├── 文档管理（上传/解析/分块/向量化/入库）
│   ├── 检索权限管理（RBAC 授权：角色/部门/用户/外部）
│   ├── API Key 管理（签发/轮换/吊销/限流配置）
│   └── 删除与保留期（软删/恢复/物理清理）
├── C. 检索与问答（授权用户/API）
│   ├── 知识库内问答（检索+生成+来源引用）
│   ├── 多轮对话上下文
│   └── 流式输出
├── D. 对外 API（OpenAI 兼容）
│   ├── /retrieve 检索
│   └── /chat 问答
└── E. 合规与可观测（贯穿全平台）
    ├── 操作审计日志
    ├── 问答留痕（QaTrace）
    ├── 数据分级字段
    └── 调用量/用量统计
```

> "六层配置"指知识库的 RAG 链路配置：数据接入策略 / 分块策略 / Embedding / 向量库 / 检索方式 / Rerank / LLM / Prompt 模板（MVP 提供一套默认配置，可后续调）。

## 4. 干系人与角色（RBAC）

### 4.1 用户画像（干系人）

| 用户类型 | 核心诉求 | 主要使用能力 |
| --- | --- | --- |
| 企业 IT / 平台管理员（系统管理员） | 统一部署、身份对接、安全合规、用户治理 | 系统管理 |
| 业务部门 / 知识库 owner（知识管理员） | 自助搭建本部门知识库问答 | 知识库管理 |
| 普通员工（内部检索者） | 自然语言获取知识 | 检索/问答（被授权后） |
| AI 应用开发者 | 将检索/问答集成到业务系统 | 对外 API（API Key） |
| 外部协作方（经认证） | 有限范围内检索授权知识库 | 对外 API / 受限界面 |

### 4.2 角色与权限矩阵

采用"用户—角色"RBAC 雏形，初始写入两角色；MVP 可单角色，后续可扩展多角色与细粒度菜单权限。

| 角色 | 系统管理 | 知识库管理 | 检索/问答 | 对外 API | 审计查看 |
| --- | --- | --- | --- | --- | --- |
| 系统管理员 | ✅ 全权 | ❌（除非兼岗） | 视授权 | 视授权 | ✅（全平台） |
| 知识管理员 | ❌ | ✅ 本租户知识库 | 视授权 | ✅（本库 Key） | 本库范围 |

**权限继承原则**：

- 知识库**默认私有**，仅创建者/同知识管理员角色可见可用。
- 检索/问答为只读使用权限，由知识管理员显式授权；**管理权不随检索授权转移**。
- 审计日志仅系统管理员/安全合规角色可见，不随知识库删除而删除。

### 4.3 默认初始数据

- 默认用户：`admin` / `123456`（首次登录强制改密）。
- 默认角色：`系统管理员`、`知识管理员`，写入角色表；`admin` 绑定两角色。

### 4.4 用户开通方式（企业内部，不开放公网注册）

1. **系统管理员创建**（MVP 默认）：在系统管理页新增用户并分配角色。
2. **企业身份同步自动开通**（可插拔，后置）：接入 SSO/OAuth2/LDAP(AD)/企业微信/钉钉/飞书后，首次登录按策略自动建号并映射角色。接入优先级：SSO/OAuth2 优先，其余依次扩展。

## 5. 关键业务流程（业务用例）

### 5.1 系统初始化与开通

```
[系统管理员] 首次登录 admin → 强制改密
    → 创建用户（如 知识管理员 A）并分配角色
    → 维护资源池（LLM/Embedding/向量库）
```

### 5.2 知识库搭建到问答闭环

```
[知识管理员] 登录 RAG 管理页
    → 创建知识库（选默认六层配置，默认私有）
    → 上传文档（PDF/MD/TXT）
    → 系统自动：解析→分块→向量化→入库
    → 授权检索对象（角色/部门/用户/外部）
    → 在库内问答（检索+生成+来源引用）
```

### 5.3 对外 API 调用

```
[知识管理员] 签发 API Key → 配置限流
[开发者] 持 Key 调用 /api/v1/chat
    → 校验 Key→定位知识库→校验隔离与检索权限
    → 检索+生成 → 返回答案（带来源）
    → 调用记录写入审计与 QaTrace
```

### 5.4 知识库删除与合规留存

```
[知识管理员] 删除知识库 → 软删除（暂停检索）
    → 保留期内可恢复
    → 过期定时任务物理清理业务数据（文档/分块/向量）
    → 审计日志 + 问答留痕 独立保留（不删）
```

## 6. 业务规则

- **检索权限（RBAC 共享）**：授权对象为内部角色 / 部门 / 具体用户 / 经认证的外部用户；权限粒度仅"检索/问答（只读）"，管理权不随授权转移；默认私有；授权/收回全程留痕。
- **数据分级联动**：知识库/文档标注敏感级别（公开/内部/机密/绝密）；高密级库仅授权且具相应密级权限者可检索。
- **资源池集中纳管**：LLM/Embedding/Rerank/向量库由系统管理员统一接入维护，知识管理员只能从资源池选择，不得自行接入外部资源（保障数据不出域）。
- **审计防篡改**：操作审计与问答留痕只读、不可删、定期归档，不随删库删除（等保取证）。
- **生命周期**：删除进入软删（暂停检索），保留期内可恢复，过期物理清理业务数据，审计/留痕独立保留。

## 7. MVP 业务范围

### 7.1 In Scope（第一个可用版本）

1. 内置 `admin` 登录系统管理页；系统管理员创建用户、角色并分配（如知识管理员）。
2. 知识管理员登录 RAG 管理页，创建知识库（命名 + 选默认六层配置）。
3. 知识库内上传文档（PDF/Markdown/TXT）→ 自动解析 → 分块 → 向量化 → 入库。
4. 知识库内问答（向量检索 + LLM 生成 + 来源引用）。
5. 检索权限 RBAC 授权（角色/部门/用户/外部），默认私有。
6. 按知识库签发 API Key + 基础按 Key 限流 + 调用审计；`/api/v1/retrieve`、`/api/v1/chat` 骨架。
7. 软删除 + 保留期 + 业务数据清理 + 审计/留痕独立留存。
8. 基础操作审计日志、问答留痕、数据分级字段、tenant_id 隔离（+基础 RLS）。
9. 私有化 Docker Compose 一键启动。

### 7.2 Out of Scope（MVP 不做）

- 配额/计费拦截（固定积分+容量+耗尽拦截，属产品化/云化能力，企业内部 MVP 不实现；仅保留字段）。
- 多数据源连接器（Notion/Confluence/DB/URL 抓取等）。
- 复杂权限/角色体系、细粒度菜单权限。
- 集群、评估平台、检索质量评估工具。
- 脱敏、密评等高级合规项（保留字段与扩展位）。
- 外部模型/外部向量库接入（统一系统管理员纳管，知识管理员不得自带）。
- 企业身份对接 SSO/LDAP/IM 的正式接入（保留可插拔扩展位，MVP 用本地账户）。

## 8. 业务验收标准（MVP）

| 编号 | 验收项 | 通过标准 |
| --- | --- | --- |
| AC-01 | 登录与角色 | admin 登录强制改密；系统管理员可创建用户并分配角色 |
| AC-02 | 知识库创建 | 知识管理员可创建知识库并选择默认六层配置，默认私有 |
| AC-03 | 文档入库 | 上传 PDF/MD/TXT 后自动解析→分块→向量化→入库，状态可见 |
| AC-04 | 问答闭环 | 库内问答返回答案且标注来源引用，支持多轮与流式 |
| AC-05 | 检索授权 | 可按角色/部门/用户/外部授权检索；未授权不可检索 |
| AC-06 | 对外 API | 持有效 Key 可调用 /retrieve、/chat；无效/吊销 Key 被拒 |
| AC-07 | 限流 | 单 Key 超速率限制被限流 |
| AC-08 | 生命周期 | 删除进入软删；保留期内可恢复；过期业务数据清理、审计/留痕保留 |
| AC-09 | 隔离 | 跨 tenant_id 数据不可互查（应用层+RLS 兜底） |
| AC-10 | 审计留痕 | 关键操作与问答写入审计/QaTrace，只读不可删 |
| AC-11 | 部署 | Docker Compose 一键启动，内网可访问 |
