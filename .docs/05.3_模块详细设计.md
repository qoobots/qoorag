## 1. 概述与范围

### 1.1 目的

将应用架构（04）的 6 个应用模块与横切关注点，落到**具体类级别**：明确类职责、分层依赖、跨类协作关系，以及关键业务流程的对象交互时序，作为编码实现与评审的依据。

### 1.2 范围

- **覆盖**：总体分层依赖、实体领域模型（11 实体 + 关联）、6 大模块协作类图、8 条关键流程时序图。
- **不覆盖**：接口请求/响应契约与错误码（见 05.4 接口设计）、逐字段数据字典（见 03 §4）。

### 1.3 模块与对应类（来自 04 §3 / 05 §5）

| 模块 | 主要类 | 职责 |
| --- | --- | --- |
| 认证鉴权（横切） | `AuthController` `AuthService` `AuthInterceptor` `WebConfig` | 登录、API Key 签发/校验、会话、拦截 |
| 系统管理 | `SystemAdminController` `UserService` `RoleService` | 用户/角色/分配、审计查看入口 |
| 知识库管理 | `RagAdminController` `KnowledgeBaseService` `ApiKeyService` | KB CRUD、检索权限、Key、生命周期 |
| 对外 API | `ApiController` | `/retrieve`、`/chat`（OpenAI 兼容） |
| 合规可观测 | `AuditService` | 审计日志、问答留痕写入 |
| 初始化 | `SeedService` | 默认租户 + 两角色 + admin 账号 |
| 横切支撑 | `common.Result` `common.SecurityContext` `common.SessionInfo` | 统一响应、ThreadLocal 安全上下文 |

> 说明：`..>` 表示依赖（非持有），`-->` 表示调用/关联。所有跨层调用最终经 Repository 落到 PostgreSQL+pgvector；外部模型服务（LLM/Embedding）当前为骨架占位，尚未被类引用。

## 3. 关键流程时序图（Sequence Diagram）

### 3.1 登录与会话建立

```mermaid
sequenceDiagram
    actor U as 浏览器/用户
    participant AC as AuthController
    participant AS as AuthService
    participant UR as UserRepository
    participant SC as SecurityContext
    participant DB as users

    U->>AC: POST /api/auth/login {username,password}
    AC->>AS: login(username,password)
    AS->>UR: findByUsername(username)
    UR->>DB: SELECT
    DB-->>UR: User(状态/密码/角色)
    AS->>AS: 校验 status=ACTIVE & deletedAt=null
    AS->>AS: passwordEncoder.matches(明文,哈希)
    AS->>AS: 构造 SessionInfo(userId,tenantId,roles,token)
    AS->>AS: sessions.put(token, info)  // 内存态
    AS-->>AC: SessionInfo
    AC-->>U: Result{ token, userId, roles }
```

> 失败分支：用户不存在 / 已停用 / 密码错误 → 抛 `RuntimeException` → 由全局异常处理（TODO）返回错误；当前 `Result.fail` 已定义但未接 `@ControllerAdvice`。

### 3.2 会话鉴权拦截（管理接口）

```mermaid
sequenceDiagram
    actor U as 调用方
    participant IN as AuthInterceptor
    participant AS as AuthService
    participant SC as SecurityContext
    participant C as SystemAdmin/RagAdminController
    participant S as Service

    U->>IN: 请求 /api/** + Authorization: Bearer <token>
    IN->>AS: validateSession(token)
    AS->>AS: sessions.get(token)
    alt 命中
        AS-->>IN: SessionInfo
        IN->>SC: set(info)
        IN-->>C: true (放行)
        C->>C: requireSysAdmin()/requireKbAdmin()
        C->>S: 业务方法
        S-->>C: Result
        C-->>U: Result
        IN->>SC: clear()  // afterCompletion
    else 未命中
        IN-->>U: 401 JSON
    end
```

> 白名单：`WebConfig` 排除 `/api/auth/login`；其余 `/api/**` 均经 `AuthInterceptor`（见 05 §8.1）。API Key 路径见 3.3。

### 3.3 API Key 鉴权（对外接口 `/api/v1`）

```mermaid
sequenceDiagram
    actor D as AI应用/外部系统
    participant IN as AuthInterceptor
    participant AS as AuthService
    participant AKR as ApiKeyRepository
    participant SC as SecurityContext
    participant AC as ApiController

    D->>IN: POST /api/v1/chat + Authorization: Bearer <apiKey>
    IN->>AS: validateApiKey(rawKey)
    AS->>AKR: findByKeyHashAndStatusAndDeletedAtIsNull(hash(rawKey),"ACTIVE")
    AKR-->>AS: ApiKey
    AS->>AS: 构造 SessionInfo(kbId,tenantId,isApiKey=true)
    AS-->>IN: SessionInfo
    IN->>SC: set(info)
    IN-->>AC: true (放行)
    AC->>AC: 读取 kbId/tenantId（检索/生成）
    Note over AC: TODO 接入 Embedding+pgvector+LLM
    AC->>AC: 写 qa_trace(kbId,tenantId,question,answer)
    AC-->>D: Result{ answer, sources }
```

> 隔离继承：API Key 绑定的 `kb_id`/`tenant_id` 注入 `SessionInfo`，后续检索/生成自动受隔离与 RBAC 约束（见 05 §8.1 / §9.3）。

```mermaid
    participant SS as SeedService
    participant TR as TenantRepository
    participant RR as RoleRepository
    participant UR as UserRepository
    participant DB as PostgreSQL

    BOOT->>SS: @PostConstruct seed()
    SS->>TR: findById(1L)  // 默认租户
    TR->>DB: SELECT
    alt 不存在
        TR->>DB: INSERT tenant(1, DEFAULT_TENANT)
    end
    SS->>RR: findByName("系统管理员"/"知识管理员")
    RR->>DB: SELECT/INSERT  // 幂等
    SS->>UR: findByUsername("admin")
    UR->>DB: SELECT
    alt 不存在
        UR->>DB: INSERT admin(tenant_id, BCrypt(pwd), 两角色)
    end
```

> 幂等：`ON CONFLICT DO NOTHING` 语义由 `orElseGet` 保证（见 05 §7 / §10）。
